## Research Questions 

Now that you are all set up, it's time to learn a little more about the tools of the trade. Edit this file and answer the following questions. You are going to 
need to start familiarizing yourself with the [GitHub docs](https://docs.github.com/en). Docs (short for documentation) are the instructions on how to use a languge or program. A large part of your job as a developer will be learning how to read and work with documentation. Please reference the GitHub docs when answering the questions below. If you cannot find what you are looking for in the docs, you can always start to practice your Google skills!

1. What is Git? Git is a open source distributed version control system 
or DVCS.

2. What is the difference between Git and GitHub?
Git is a program than is open source and anyone can use for free for 
version control of their projects.  Github is a company that works with 
git to store your projects versions in the cloud server.

3. Why do we create a branch?
So that we are working on an independent line of code for working new 
code or fixing bugs before merging back to the "main" line of code.
 
4. What is the purpose of a Pull Request?
 To make sure you are always working on the latest version from the 
remote repository.

5. What is the command you can use to switch between branches? For example you are working on FIRSTNAME-LASTNAME branch and you want to switch back to main.
git checkout

6. Explain the difference between `git fetch`, `git merge` and `git pull`. What does each command do?
Git Fetch just downloads from repo
Git Pull fetchs and integrates with repo or local
Git merge combines two or more branchs together

7. What is a merge conflict?
A merge conflict is when two developers made changes to the same line of code in the same part of the project.

8. How do you resolve a merge conflict?
There are two way to resolve a conflict. One Decide not to merge or resolve the conflict with tools from git. The tools provided will show you the diff from 
every branch and find out how you can resolve it.
